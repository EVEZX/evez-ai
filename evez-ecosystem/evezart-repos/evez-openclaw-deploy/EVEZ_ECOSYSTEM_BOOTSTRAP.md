# EVEZ Ecosystem Bootstrap for OpenClaw

This deployment now seeds the OpenClaw workspace with Steven Crawford-Maggard / EvezArt identity and repo context so the gateway recovers as an EVEZ-aware assistant instead of a blank OpenClaw instance.

## What is seeded

- `workspace/EVEZ_IDENTITY.md` — EVEZ mission, accounts, and key repos.
- `workspace/ecosystem/github-public-repos.json` — public EvezArt repo inventory from the GitHub API.
- `workspace/ecosystem/github-public-repos.md` — human-readable repo/context bootstrap.
- `workspace/skills/clawdhub` — verified Clawdhub skill from ClawHub.
- `workspace/skills/lobster` — verified Lobster skill from ClawHub.
- OpenClaw config generated by `scripts/evez-openclaw-onboard.sh` enables the bundled, host-compatible `lobster` plugin.

## Local private repo inventory

If `GITHUB_TOKEN` or `GH_TOKEN` is present in the local OpenClaw env, run:

```bash
OPENCLAW_STATE_DIR=~/.openclaw ./scripts/evez-openclaw-ecosystem-sync.sh --include-private
```

This writes an authenticated inventory to:

```text
~/.openclaw/workspace/private/github-visible-repos-private.json
```

That path is ignored by git and should stay local-only. It can include private repository metadata.

## Public refresh

To refresh only public context:

```bash
./scripts/evez-openclaw-ecosystem-sync.sh --public-only
```

## Lobster path used

The literal command `openclaw plugins install clawhub:@openclaw/lobster` is not supported by the current local OpenClaw host (`2026.1.30`); the plugin installer falls through to npm and rejects the `clawhub:` protocol.

The safe host-compatible setup is:

1. Install verified ClawHub `lobster` skill (`guwidoe/lobster`, version `1.0.1`).
2. Install the verified CLI package `@clawdbot/lobster`.
3. Enable the bundled OpenClaw `lobster` plugin that matches the host (`2026.1.30`).

Latest npm `@openclaw/lobster` requires a newer OpenClaw host/plugin API; do not force it into this host until OpenClaw is upgraded.
